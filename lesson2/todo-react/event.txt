Состояние. 
ОБЫЧНАЯ переменная в компоненте живет один раз, при первом рендере, но при следующем обновлении интерфейса react 
пересоздаст компонент и все изменения потеряются
Состояние хранится внутри самого react. При изменении react сам запускает перерисовку компонента с обновленными данными,
нам не надо руками обновлять интерфейс

useState
- хук для управления состоянием в функциональных компонентах
- возвращает массив [текущее значение, функция для обновления]
- синтаксис: const [state, setState] = useState(начальное значение)
- при вызове setState React запускает перерендер компонента
- каждый вызов useState создает отдельное состояние
- необходимо импортировать из react: import { useState } from 'react'
